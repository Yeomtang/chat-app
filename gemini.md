# 프로젝트 개발 로그 (Gemini.md)

본 문서는 실시간 익명 채팅 및 LED 전광판 시스템의 개발 단계를 기록하고 추적하는 문서입니다.

## 🔗 배포 주소 (Render)
* **관객용 채팅 화면**: [https://chat-app-6kl5.onrender.com/](https://chat-app-6kl5.onrender.com/)
* **LED 전광판 화면**: [https://chat-app-6kl5.onrender.com/led](https://chat-app-6kl5.onrender.com/led)
* *주의: Render 무료 플랜 특성상 최초 접속 시 서버가 깨어나는 데 약 30초~1분 정도의 지연이 발생할 수 있습니다.*

---

## 📅 개발 단계 및 진행 상황

### ✅ 1단계: 실시간 익명 채팅창 개발 (완료)
* **기능 구현**:
  * QR코드 인식 접속을 가정한 모바일 웹 채팅창 구현
  * 직장인 피로 테마 닉네임 자동 배정 시스템 (625개 풀)
  * iMessage 스타일의 말풍선 UI 및 자동 스크롤 기능
  * 실시간 LED 전광판 전용 화면 구현 (최대 8개 메시지 노출, 상단 페이드아웃 효과)
* **버그 수정**:
  * 한글(CJK) 입력 시 `Enter` 키 입력 과정에서 메시지가 두 번 반복 전송(자음/모음 분리 전송)되는 IME 조합 버그 해결 (`e.isComposing` 감지 적용)
* **배포**:
  * GitHub 연동 및 Render Cloud를 통한 웹 서비스 배포 완료

### ⬜ 2단계: 객관식 (Yes/No) 투표 & 공 애니메이션 (대기)
* **상세 기능**:
  * 제작진 트리거 시 질문 팝업 등장 및 30초 타이머 작동 (LED + 모바일 동시 노출)
  * 투표 시 YES/NO 영역으로 작은 원(공)들이 날아가 쌓이는 실시간 물리 애니메이션
  * 30초 제한 시간 동안 답변 변경 허용 및 실시간 비율 변화 표현
  * 타이머 종료 후 최종 투표 결과 고정 기능

### ⬜ 3단계: 주관식 질문 & 워드클라우드 (대기)
* **상세 기능**:
  * 제작진 트리거 시 주관식 질문 팝업
  * 개별 답변 흘려보내기(스크롤 애니메이션) 또는 실시간 키워드 추출을 통한 워드클라우드 시각화

### ⬜ 4단계: 제작진 관리자 인터페이스 (대기)
* **상세 기능**:
  * 사전 질문 리스트 작성 및 순서 편집 기능
  * 방송 진행 상황에 맞춘 실시간 트리거 제어 (채팅 모드 ↔ 질문/투표 모드 전환)
  * 방송 중 즉흥적인 질문 추가 및 즉각 적용 기능

### ⬜ 5단계: 디자인 디테일 및 LED 최적화 (대기)
* **상세 기능**:
  * 실제 방송 LED 화면 크기 및 가시성 맞춤 폰트/색상 디테일 튜닝
  * LTE 모바일 접속 안정성 테스트 및 최종 버그 수정

## 🛠️ 기술 스택 및 환경
* **Backend**: Node.js, Express, Socket.io
* **Frontend**: HTML5, Vanilla CSS, Vanilla JavaScript
* **Database**: 메모리 내 저장 (최근 50개 채팅 히스토리 저장)
* **Deployment**: GitHub + Render (Free Web Service)
