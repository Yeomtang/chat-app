# 프로젝트 메모리

## 프로젝트 개요
방송 제작 배경 LED 화면용 실시간 익명 채팅 시스템

## 핵심 요구사항
- **용도**: 방송 배경 LED 화면에 상시 노출되는 채팅창
- **참여자**: 현장 토론자 + 관객 (익명 채팅)
- **입장 방식**: QR코드 스캔 → 웹페이지 입장 (앱 다운로드 없음)
- **네트워크**: LTE 환경에서도 원활하게 동작해야 함 (Wi-Fi 의존 X)
- **채팅 UI**: iMessage 스타일, 메시지가 위로 계속 올라오는 구조
- **익명성**: 관객은 익명으로 참여

## 화면 구성
- 비율: 2:3 세로형 (핸드폰 세로 비율 기준, LED 특성 반영)
- 화면 수: 1개
- 테마: 라이트모드
- 채팅 스타일: 깔끔한 말풍선, 좌측 정렬 (첨부 이미지 참고)
- 입장 시 닉네임: **직장인 피로 테마 랜덤 닉네임 자동 추천**
  - 예: "야근 3년차", "퇴근하고 싶다", "월요일이 싫은 사람" 등
  - 닉네임은 LED 화면 말풍선에도 표시됨

## 화면 모드 (2가지)
1. **채팅 모드 (기본)**: 채팅창이 전체 화면 차지, iMessage 스타일로 위로 흐름
2. **질문 모드 (제작진 트리거)**: 채팅창 블러 처리 → 공통 질문 팝업 등장
   - 예시 질문: "지금 본인의 회사 생활에 만족하십니까?"
   - 답변 형식 2가지:
     - **객관식 (Yes/No)**: 실시간 비율 시각화
       - YES 원 / NO 원 두 개의 원형 클러스터
       - 투표마다 작은 원(공)이 해당 원으로 날아들어오는 애니메이션
       - 공들이 반대쪽 원으로 이동하는 흐름으로 실시간 비율 변화 표현
       - 답변 변경 가능, 단 **30초 타이머** 제한
       - 30초 카운트다운: **LED 화면 + 관객 핸드폰 양쪽에 표시**
       - 타이머 종료 후 최종 결과 고정
       - 흐름: 질문 등장 → 30초 카운트다운 → 실시간 공 이동 → 종료 → 결과 확정
     - **주관식 (자유 텍스트)**: 개별 답변 흘려보내기 or 워드클라우드

## 관리자 인터페이스 (제작진용)
- 질문을 **미리 여러 개 작성**해두고 순서대로 실행
- 기능: 질문 입력/편집 → 순서 정렬 → 시작 버튼 → 결과 확인
- 방송 중 즉흥적으로 질문 추가도 가능해야 할 듯

## 전체 화면 흐름
1. **채팅 모드**: iMessage 스타일 익명 채팅, 위로 스크롤
2. **질문 모드 진입** (제작진 트리거): 채팅 블러 → 질문 텍스트 + 30초 타이머 등장
3. **투표 중**: YES/NO 공 애니메이션 실시간, 타이머 LED+핸드폰 동시 표시
4. **타이머 종료**: 질문 텍스트·타이머 사라짐 → **결과(원형 클러스터)만 화면에 floating**
5. **결과 유지**: MC/토론자 코멘트 시간, 제작진이 수동으로 다음 전환
6. **채팅 모드 복귀** (제작진 수동 트리거): 다시 익명 채팅, 관객이 토론자에게 질문/의견 전송

## 미결 사항 / 논의 중
- ✅ 질문/투표 모드 중 채팅 입력 완전 차단 → 오직 답변에만 집중
- ✅ 투표 후 핸드폰 화면: 내가 선택한 쪽 하이라이트 + 실시간 비율 표시
- 채팅 모드 ↔ 질문 모드 전환 시 LED 화면 애니메이션
- 주관식 결과 표시 방식 세부 (흘려보내기 vs 워드클라우드 선택)

## 진행 상황 (구현 단계)
- ✅ **1단계 완료 + 배포**: 채팅 시스템 (QR 입장 가정, 닉네임 자동배정 625개 풀, iMessage 스타일, LED 표시)
  - 관객: https://chat-app-6kl5.onrender.com/
  - LED: https://chat-app-6kl5.onrender.com/led
  - ⚠️ Render 무료 플랜은 비활성 시 서버 슬립 → 첫 접속 시 30~60초 지연 가능. 실사용(12월) 전 유료 플랜 전환 검토 필요
- 🔄 **2단계 구현 중**: Yes/No 투표 + 공 애니메이션 + 30초 타이머
  - 서버(server.js): `mode: 'chat'|'voting'|'result'` 상태머신 추가. `votes` 객체는 clientId → 'yes'|'no'로 저장(중복 방지). `identify` 이벤트로 클라이언트 영구 ID 수신. `admin:startVoting/endVoting/returnToChat/getState` 이벤트로 제작진 제어. 타이머는 서버가 `votingEndTime`(epoch ms)을 계산해 브로드캐스트, 각 클라이언트는 로컬에서 카운트다운만 표시(서버 권위 방식).
  - LED 화면(led.html): 채팅 레이어 블러 처리 + 투표 오버레이. Canvas 기반 공 파티클 애니메이션(클라이언트별 1개 공, YES/NO 클러스터로 sunflower 패킹 배치, 부드러운 이동 애니메이션). voting 모드에서 질문+타이머 표시, result 모드에서는 질문/타이머 숨기고 결과 비율만 floating 표시.
  - 관객 화면(chat.html): localStorage 기반 영구 `clientId` 생성 → 소켓 연결 시 `identify`로 서버에 전송(재접속해도 동일 투표자로 인식). 투표 오버레이(질문+타이머+YES/NO 버튼+실시간 비율 바) 추가, 투표 중엔 채팅 입력 완전 차단(블러 처리). 투표 후 내가 선택한 버튼 하이라이트 유지.
  - 임시 관리자 화면(admin.html): 정식 4단계 화면 전까지 사용할 임시 컨트롤 패널. 현재 상태 모니터링(모드/질문/YES·NO 카운트/남은 시간), 질문 사전 등록(localStorage 저장, 순서대로 실행), 즉석 질문 시작, 투표 강제 종료, 채팅 모드 복귀 기능 포함.
  - ⬜ 남은 작업: 로컬 테스트 → 사용자 확인 → Render 재배포
- ⬜ 3단계: 주관식 질문 + 워드클라우드
- ⬜ 4단계: 제작진 관리 화면 (질문 사전 등록, 순서 실행) — 임시 버전은 2단계에서 선반영됨, 정식 디자인/기능 보강 필요
- ⬜ 5단계: 디자인 디테일 + LED 최적화
